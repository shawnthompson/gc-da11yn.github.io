Testing

QA testers who validate accessibility conformance through automated and manual testing, ensuring products meet accessibility requirements and work with assistive technologies.

Related roles:

  • Automated QA Testing
  • Manual QA Testing

Testing and Accessibility

QA Testers run automated test frameworks or manually test products to confirm correct operation based upon provided requirements. The main accessibility role of a QA Tester in the accessibility lifecycle is to understand the accessibility requirements that exist and to run tests to ensure the product or feature conforms to those requirements.

QA Testing Overview

Quality Assurance (QA) Testers typically don't contribute directly to the design and development phases of a product. They may have the opportunity to review and sign off on designs before they are implemented. The testing roles include both automated and manual testing approaches, and many QA professionals will do both.

Automated QA Testing

Quality Assurance automation frameworks run against products to test features and functionality. QA Testers performing automated tests are responsible for running tests within an automation framework that covers accessibility features and requirements.

Automated Testing Responsibilities:

  • Implementing automated accessibility testing tools in CI/CD pipelines
  • Running automated accessibility scans using tools like axe, WAVE, or Lighthouse
  • Setting up accessibility testing frameworks and libraries
  • Creating automated tests for keyboard navigation and focus management
  • Integrating accessibility checks into existing test suites
  • Monitoring accessibility regression through automated testing

Note: Automated tools typically uncover about 30-35% of potential accessibility issues. The remaining 65-70% require manual testing.

Manual QA Testing

Manual QA Testing handles testing that cannot be covered through automated tools. These testers typically have deeper knowledge about accessibility requirements and experience operating assistive technologies.

Manual Testing Responsibilities:

  • Testing with screen readers (NVDA, JAWS, VoiceOver, TalkBack)
  • Validating keyboard-only navigation and interaction
  • Testing color contrast and visual accessibility
  • Verifying proper heading structure and semantic markup
  • Testing form accessibility and error handling
  • Validating ARIA implementation and assistive technology compatibility
  • Testing across different devices and assistive technology combinations
  • Conducting usability testing with users who have disabilities

Testing Process

  1. Requirements Review: Understanding accessibility requirements and acceptance criteria
  2. Test Planning: Creating comprehensive test plans that include accessibility scenarios
  3. Automated Testing: Running automated accessibility scans and tests
  4. Manual Testing: Conducting detailed manual accessibility testing
  5. Assistive Technology Testing: Testing with actual assistive technologies
  6. User Testing: Including users with disabilities in testing processes
  7. Reporting: Documenting findings with clear remediation guidance
  8. Verification: Retesting after fixes are implemented

Common Job Titles

Automated Testing:

  • QA Automation Engineer
  • Quality Engineer
  • Automation Engineer
  • Test Automation Developer

Manual Testing:

  • QA Analyst
  • QA Specialist
  • User Acceptance Tester
  • Functional Tester
  • Accessibility Tester
  • Usability Tester

Key Deliverables

  • Automated accessibility test suites and frameworks
  • Manual accessibility test cases and scripts
  • Accessibility testing reports with detailed findings
  • Test documentation and procedures
  • Accessibility defect reports with remediation guidance
  • Test results validation and verification reports
  • Assistive technology compatibility matrices
  • User testing reports including accessibility insights

Testing Tools and Technologies

Automated Tools:

  • axe-core and axe browser extensions
  • WAVE Web Accessibility Evaluation Tool
  • Lighthouse accessibility audits
  • Pa11y command line accessibility tester
  • Selenium with accessibility testing libraries

Assistive Technologies:

  • Screen readers (NVDA, JAWS, VoiceOver, TalkBack)
  • Voice recognition software (Dragon NaturallySpeaking)
  • Switch navigation devices
  • Screen magnification software
  • Alternative keyboards and input devices
